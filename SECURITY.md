# Security

## Secrets

Never expose admin secrets, JWT secrets, R2 keys, signing keys, email keys, or service credentials through frontend code or `VITE_` variables.

## Authorization

Authorization is enforced at four layers:

1. PostgreSQL and Hasura permissions
2. Worker endpoint checks
3. GraphQL query filters
4. Frontend route and action visibility

Frontend hiding is not considered security.

## Authentication

Nhost authentication is the production path. Local development fallback is clearly marked and stores state in `sessionStorage` only.

## Sessions

The frontend defaults to a 15-minute inactivity timeout controlled by `VITE_SESSION_TIMEOUT_MINUTES`.

## Files

R2 buckets must remain private. Preview and download access must use short-lived authorized URLs generated by Workers.
